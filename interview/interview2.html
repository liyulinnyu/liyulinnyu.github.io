4. BFC (Block Formatting Contexts) Block Formatting Contexts is an isolated rendering area on the page, and the child elements inside the container do not affect the layout of the external elements. Float cannot be none, overflow cannot be visible, position cannot be relative and static, display should be one of [table-cell, table-caption, inline-block]. 5. IFC (Inline Formatting Contexts) The height of the line box (line frame) of the IFC is calculated from the highest actual height contained in the elements within the row (not affected by the vertical padding/margin) 6. FFC (Flex Formatting Contexts) An element with a display value of flex or inline-flex will generate an adaptive container (Flex, container) 7. Picture lazy loading Principle: If page contains too many pictures and we want to load them at once, it would be cost amounts of time. So the lazy load method is to delay loading when the pictures aren't in visual page. In order to achieve this method, we should use "data-" property instand of "src", and when showing pictures we will let the "src" be "data-". Before that, we can use an array to save all "data-" properties and reduce workloads. There has a ready-made package you can use echo.js 8. Page loading progress bar Unluckily, there has no method to give us the number of loaded data. So we have to simulate the loading progress bar. How to simulate? We should insert some javascript blocks into our html codes. And increase the progress bar based on the number of loaded html codes.
